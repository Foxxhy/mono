<h2 id="quand-utiliser">Quand utiliser</h2>

Composant de la famille **Structure**. Utilisez les classes du bundle `dist/design-system.css` et gardez le HTML natif autant que possible.

<h2 id="html-minimal">HTML minimal copiable</h2>

```html
<div class="table-wrapper">
  <table class="table table-striped">
    <caption class="caption">Utilisateurs</caption>
    <thead><tr><th scope="col">Nom</th><th scope="col">Rôle</th></tr></thead>
    <tbody><tr><td>Ada</td><td>Admin</td></tr></tbody>
  </table>
</div>
```

<h2 id="variantes">Variantes</h2>

Les variations visuelles sont disponibles dans l’onglet stories de cette page (`Components/Table`). Les classes proviennent de `packages/design-system/src/components/table.scss`.

<h2 id="rgaa">RGAA et accessibilité</h2>

- Utiliser un vrai `<table>` pour des données tabulaires, avec `<caption>` si utile et `scope="col"`/`scope="row"`.
- Conserver un ordre de focus logique et visible.
- Ne pas transmettre une information uniquement par la couleur.
- Préférer les éléments HTML natifs avant d’ajouter des rôles ARIA.

<h2 id="à-éviter">À éviter</h2>

- Remplacer un élément natif interactif par une `div` cliquable.
- Supprimer le focus visible ou le libellé accessible.
- Copier les couleurs hexadécimales : utiliser les classes et tokens du design system.

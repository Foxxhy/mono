<h2 id="quand-utiliser">Quand utiliser</h2>

Composant de la famille **Navigation**. Utilisez les classes du bundle `dist/design-system.css` et gardez le HTML natif autant que possible.

<h2 id="html-minimal">HTML minimal copiable</h2>

```html
<footer class="footer">
  <div class="footer-content">
    <p class="footer-text">Nom du service</p>
    <nav class="footer-links footer-links-row" aria-label="Liens de pied de page">
      <a class="footer-link" href="/accessibilite">Accessibilité</a>
    </nav>
  </div>
</footer>
```

<h2 id="variantes">Variantes</h2>

Les variations visuelles sont disponibles dans l’onglet stories de cette page (`Components/Footer`). Les classes proviennent de `packages/design-system/src/components/footer.scss`.

<h2 id="rgaa">RGAA et accessibilité</h2>

- Le footer doit rester identifiable avec `<footer>`; les groupes de liens utilisent `<nav aria-label="…">`.
- Conserver un ordre de focus logique et visible.
- Ne pas transmettre une information uniquement par la couleur.
- Préférer les éléments HTML natifs avant d’ajouter des rôles ARIA.

<h2 id="à-éviter">À éviter</h2>

- Remplacer un élément natif interactif par une `div` cliquable.
- Supprimer le focus visible ou le libellé accessible.
- Copier les couleurs hexadécimales : utiliser les classes et tokens du design system.

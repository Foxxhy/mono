<h2 id="quand-utiliser">Quand utiliser</h2>

Composant de la famille **Surfaces**. Utilisez les classes du bundle `dist/design-system.css` et gardez le HTML natif autant que possible.

<h2 id="html-minimal">HTML minimal copiable</h2>

```html
<article class="card card-border">
  <h2 class="heading-md">Titre de carte</h2>
  <p class="body-sm u-mt-md">Contenu court et groupé.</p>
</article>
```

<h2 id="variantes">Variantes</h2>

Les variations visuelles sont disponibles dans l’onglet stories de cette page (`Components/Card`). Les classes proviennent de `packages/design-system/src/components/card.scss`.

<h2 id="rgaa">RGAA et accessibilité</h2>

- Une carte ne remplace pas un lien ou un bouton : si toute la carte est cliquable, son nom accessible doit être clair.
- Conserver un ordre de focus logique et visible.
- Ne pas transmettre une information uniquement par la couleur.
- Préférer les éléments HTML natifs avant d’ajouter des rôles ARIA.

<h2 id="à-éviter">À éviter</h2>

- Remplacer un élément natif interactif par une `div` cliquable.
- Supprimer le focus visible ou le libellé accessible.
- Copier les couleurs hexadécimales : utiliser les classes et tokens du design system.
